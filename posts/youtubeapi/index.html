April - May 2020

For my final project in my Social Media and Data Mining class, I wanted to evaluate the toxcity of YouTube communities by performing sentiment analysis on YouTube comments.

What was our data mining process?

Storing the Data: MongoDB

Because our team was working remotely and needed to share the data for analysis and examination of results, we decided to use a cloud-based database. We knew our data would primarily be stored in the form of JSON documents, and non-relational databases work very well with JSON documents. This criteria led us to choose the MongoDB database.

Mining the Data: YouTube API

We broke the data mining process into 2 parts:

  1. We started by collecting a list of the most popular videos from YouTube. We retrieved the most popular YouTube videos over the course of 5 days (April 2nd - 8th).
  2. Then we collected the comments for each of the videos in the list. We had to automate the comments mining process in order to circumvent the limit placed on API calls.

Toxicity Analysis Algorithm: Perspective

When designing our toxicity analysis algorithm, we referenced these two research papers: “Identification and Classification of Toxic Comments on Social Media using Machine Learning Techniques” (Ozoh et al) and “Developing an online hate classifier for multiple social media platforms” (Salminen et al). These two papers helped us understand the textual and sentiment analysis techniques that are leveraged in order to analyze the toxicity of online social interactions, which we found to include the Term Frequency-Inverse Document Frequency technique, Logistic regression algorithm, and Bag of Words technique.

We intended to use a toxicity analysis model called bag-of-communities developed by Eshwar Chandrasekharan, Mattia Samory, Anirudh Srinivasan, and Eric Gilbert. However, the model was written in Python2 and mostly depracated. Due to limited time and resources, we decided to use Perspective API. Although backed by a large technology company, Perspective API is a black box. Perspective API is a toxicity analysis model developed by researchers at Google that uses various machine learning models. Despite not being able to understand its inner-workings, we felt that the API suited the scope of our project and proceeded on.
